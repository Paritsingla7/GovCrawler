User Guide

What is GovCrawler?

GovCrawler is a local lead-generation and email outreach platform for Indian government domains. It crawls .gov.in and .nic.in websites, extracts official email addresses and personnel information, and lets you send personalised outreach emails — all from a single web interface running on your machine.

There are two phases:

  • Phase 1 — Lead Generation: Import government domains → select and crawl → review extracted contacts.
  • Phase 2 — Email Outreach: Configure SMTP → create templates → build campaigns → dispatch and monitor.

Step 1 — Import Domains

Before crawling anything you need to load the government domain registry. The Dashboard sidebar offers two methods:

JSON Upload (Recommended)

Click Import next to "Choose a JSON file…" and upload a gov_domains.json export from the india.gov.in web directory. This is instant — no API calls, no internet needed. A progress bar appears while the file is processed; the domain count in the top-right header updates when done.

Tip: Download the JSON file once and reuse it across sessions. The file does not expire. Use Live API Refresh only when you need the very latest domain additions.

Live API Refresh

Click Refresh to fetch the current domain list directly from india.gov.in. This requires an internet connection and takes longer on first use. Use it when your local JSON is outdated or when newly registered government domains are needed.

Note: Both methods replace the existing domain list in full. Clicking Refresh while an import is already running shows "Import already running" — wait for the first to complete.

Filters on the Dashboard

Once domains are loaded, use the sidebar filters to narrow the list before selecting domains to crawl:

  • Category — Government tier (Central Government, State Government, District, etc.)
  • State — Restricts to domains for a specific state. Updates automatically when a Category is chosen.
  • Org Type — Department, Board, Commission, Tribunal, etc.
  • Search — Free-text match against domain name or ministry title.

The count label below the filters shows how many domains match the current selection. Click Clear Filters to reset all dropdowns at once.

Step 2 — Run a Crawl Job

After filtering to the domains you want, select them using the checkboxes in the domain table:

  • Select page — Selects all 50 domains visible on the current page.
  • Select all N — Selects every domain matching your filters, even across pages. This button appears after you first check a page.
  • Clear — Deselects everything.

Once at least one domain is checked, the ▶ Crawl Selected button appears in the toolbar. Click it to start a crawl job. The bottom dock opens automatically and the job begins running in the background.

Recrawl Protection

GovCrawler tracks every URL it visits. By default, any URL visited in the last 30 days (across all jobs) is skipped on re-runs. To force a full re-crawl, go to Settings and click Clear Cache under Maintenance, or set Recrawl Days to 0.

Pagination-Aware Crawling

Long listings — a 30-page staff directory, a multi-page "Who's Who" — normally get cut off early: every link, including "Next page →", counts against Max Depth the same as any other link, so the crawler runs out of depth budget long before it reaches the last page.

With pagination-aware crawling on (Settings → Advanced, enabled by default), a genuine "Next page" link — a numbered page link, rel="next", or text like "Next" / "»" / "Last" — advances a separate counter instead of Max Depth, so the crawler follows the whole chain to completion. A page is only recognized as pagination if it's clearly one: the detector is deliberately conservative so it can't be tricked by a site whose "Next" link actually leads into an endless loop of session/tracking URLs — those still get bounded by Max Depth exactly as before, no matter how many pages the crawler follows via pagination.

Max Pages per Chain is the backstop even if a chain is misjudged: it hard-stops after that many "Next" hops regardless. Max Children per Chain caps how many ordinary (non-"Next") links the WHOLE chain may pull in combined, so a 50-page chain can't enqueue 50× a single page's usual link budget.

Step 3 — Review & Export Leads

Navigate to Leads to see all extracted contacts across every crawl job. The sidebar filters narrow the results; the main table shows each contact's email, name, designation, department, and source.

Understanding the Band Column

  • HIGH (green) — Email found in a structured table row or a mailto: link, with a person name identified nearby. High confidence.
  • LOW (yellow) — Email found via proximity scan (regex pattern). Name or designation may be inaccurate or absent.

A yellow left-border on a row means one or more fields (name, designation, or department) are missing. These rows are automatically deselected when you create a campaign so they won't receive a badly rendered email.

Inline Editing

Click any editable cell in the table (Name, Designation, Department, State) to edit it in place. Press Enter or click elsewhere to save. The change persists immediately — no separate Save button needed.

Filters & Search

  • Job — Show leads from one specific crawl job only.
  • Category / State — Geographic and tier-based filtering.
  • Search — Matches against name, email, and designation fields.
  • Complete data only — Hides leads with no name or no designation. Use this before building a campaign to see only well-populated records.

CSV Export

Click ⬇ Export CSV (top right of the Leads page) to open the field selector. Email is always included. Tick or untick any other column:

Name  ·  Designation  ·  Department  ·  Domain/Org  ·  State  ·  Org Type  ·  Category  ·  Source URL  ·  Page Title  ·  Context Snippet  ·  Captured At  ·  Crawl Depth

The export respects your current filters and selection. If leads are checked, only those N leads are exported. If nothing is selected, all leads matching the current filters are exported.

Step 4 — Outreach Setup

Before creating campaigns, configure three things in Settings:

SMTP Credentials

Go to Settings → SMTP Credentials → + Add Credential. Fill in:

  • Host — Your SMTP server address (e.g. smtp.gmail.com)
  • Port — 465 for SSL/TLS, 587 for STARTTLS
  • Username — Your sender email address
  • Password — Your app password (not your regular account password)

Gmail users: Generate an App Password in your Google Account → Security → 2-Step Verification → App Passwords. Your regular Google password will not work and will cause an auth failure.

You can add multiple credentials. During dispatch, GovCrawler uses them in round-robin rotation — each email is sent through the next available credential, spreading the load across sender accounts.

Credential states:

State What it means What to do
Active Available for sending Nothing — it will be used automatically
Cooling Temporarily paused after a rate-limit error (421 / 450 / 451). Cooldown duration shown in the table. Wait — it auto-resumes after cooldown expires
Disabled Auth failure — the SMTP server rejected the username or password Fix the password, then re-enable manually via the Edit action

Use Test Selected to verify credentials connect successfully before running a real campaign. A successful test auto-activates the credential.

Email Templates

Go to Settings → Email Templates → + Create Template. Templates use Jinja2 syntax for dynamic content per recipient.

Available variables:

Variable Resolves to Fallback if missing
{{ name }} Lead's person name Subject: "Official"  /  Body: [MISSING: name]
{{ designation }} Lead's designation / job title Subject: blank  /  Body: [MISSING: designation]
{{ email }} Lead's email address Always present

Example template:

Subject: Important Communication — {{ designation }}, {{ name }}

Dear {{ designation }} {{ name }},

We are writing to inform you about an upcoming initiative
that may be relevant to your role.

Please feel free to reach us at [Your Contact].

Regards,
[Your Name]

Tip: Use the Insert chip buttons in the template editor to add variables at the cursor position. The template is validated for Jinja2 syntax on save; if there is an error, the line number and message are shown.

Blacklist

The blacklist prevents emails from being staged in new campaigns and from being sent in existing ones. Addresses are added two ways:

  • Automatically — When an email hard-bounces (SMTP codes 550 or 553), GovCrawler adds it to the blacklist and marks the email FAILED.
  • Manually — Settings → Blacklist → + Block Email. Use this for opt-out requests or known-invalid addresses.

To unblock an address, click the delete icon on its row in the blacklist table. It will be eligible for future campaigns again.

Step 5 — Create a Campaign

Campaigns turn a selection of leads into email drafts ready to send. The system auto-renders one draft per lead before you dispatch anything.

Creating from the Leads Page

  1. On the Leads page, use filters to find the contacts you want to target.
  2. Check the leads you want, or click Select all N for the full filtered set.
  3. Click + Create Campaign in the toolbar.
  4. Enter a Campaign Name and choose a Template from the dropdown.
  5. Click Generate Drafts. You are taken to the Campaigns page.

To add leads to an existing campaign later, select them and click + Add to Campaign instead, then pick the target campaign from the dropdown.

Reviewing Drafts

The campaign opens with status PAUSED. No emails have been sent. The email table shows every draft:

  • Checkbox — Checked = will be sent; Unchecked = skipped. Leads with missing data are auto-unchecked on creation.
  • Recipient / Subject — Preview the rendered output. [MISSING: name] in the subject means that lead has no name.
  • Edit icon — Opens a modal to manually override the subject and body for that specific recipient only.
  • Delete icon — Removes the draft from the campaign entirely.

Always review drafts before dispatching. Fix or uncheck any draft containing [MISSING:] markers — these create a poor first impression when sent.

Step 6 — Dispatch & Monitor

On the Campaigns page, select a campaign from the sidebar and click ▶ Dispatch emails. A confirmation dialog shows the count of checked drafts. Click Yes, Dispatch to begin.

Rate limiting is intentional. GovCrawler waits 30–90 seconds between sends (randomised) to avoid triggering SMTP rate limits. A batch of 100 emails takes roughly 50–90 minutes. Do not close the app mid-dispatch.

Pause and Cancel

  • Pause — Stops dispatching after the current send finishes. Status returns to PAUSED. Click Dispatch again to resume — it picks up exactly where it stopped.
  • Cancel — All remaining QUEUED emails are marked FAILED. This cannot be undone. Use only to abandon a campaign entirely.

Auto-Blacklist on Hard Bounce

If a recipient's mail server permanently rejects an email (SMTP codes 550 or 553), GovCrawler automatically:

  1. Adds the address to the blacklist (Settings → Blacklist).
  2. Marks the email as FAILED in the campaign with the bounce reason in the Error/Note column.
  3. Excludes the address from all future campaigns automatically.

You can remove a blacklist entry manually if the address was corrected or if the block was accidental.

Step 7 — Test Campaigns

Before sending to real leads, validate that your SMTP credentials and email template work correctly. Go to Settings → Run Test Campaign, or navigate directly to /test-campaign.

Fill in the form:

  • Campaign Name — An internal label (e.g. "SMTP Check 1")
  • Select Template — The template you want to preview
  • Use Round Robin — Checked = send through all active credentials in rotation. Uncheck to test one specific credential.
  • Dummy Lead Details — Add Name, Designation, Department, and your own email address as the recipient. Use "+ Add Another Lead" to test multiple slots.

Click Create & Dispatch Test. The test campaign is created and dispatched immediately. You should receive the test email within a minute or two.

What to verify when the email arrives: (1) It arrived in your inbox (not spam). (2) Template variables rendered correctly — no [MISSING:] markers. (3) Subject and body look as intended. (4) Sender address is the credential you expected.

To review test results, go to Campaigns and enable the Show Tests checkbox in the sidebar. Test campaigns appear alongside production campaigns.

Settings Reference

All settings are stored in portal/config.yaml. The Settings page lets you edit most of them in the browser. A quick-access panel (⚙️ button on the Dashboard header) covers the most common crawler knobs without leaving the dashboard.

Important: Crawler settings take effect on the next job you create. Any job currently running continues with its original settings.

Crawler Settings

Setting Description
Workerscrawler.workersDefault: 50 Number of parallel coroutines crawling simultaneously. Reduce to 10–20 for slow machines or rate-sensitive targets. Higher values increase throughput but also put more load on crawled servers.
Max Depthcrawler.max_depthDefault: 3 How many link hops from the seed URL to follow. 0 = seed page only (fastest, fewest leads). 3 = enough to reach most /contact, /about, /staff pages. Beyond 4 grows exponentially in URL count.
Recrawl Dayscrawler.recrawl_daysDefault: 30 URLs visited in any job within the last N days are skipped globally. Set to 0 to always re-crawl everything (useful for testing). Use Clear Cache in Settings to reset the history without changing this number.
Request Delaycrawler.request_delayDefault: 1.5 s Minimum seconds between consecutive requests to the same domain (politeness throttle). Increase to 3–5 s if a site starts returning 429 errors or blocking your IP address.
Per-URL Timeoutcrawler.per_url_timeoutDefault: 100 s Hard watchdog per URL. If fetch + parse takes longer, the worker skips it and moves on. Must be higher than the Playwright timeout to avoid internal conflicts.
Try HTTP Firstcrawler.httpx_firstDefault: true Attempt a lightweight HTTPX request before launching a browser. Most .gov.in sites are static HTML — this saves significant time and resources. Disable only if you want Playwright for every URL (not recommended).
Playwright Fallbackcrawler.playwright_fallbackDefault: true If a page contains JavaScript-rendering indicators (React/Next.js root elements, "Please enable JavaScript"), escalate to a full Chromium browser. Disable to skip JS-heavy pages entirely — faster but misses some modern government portals.
Playwright Timeoutcrawler.playwright_timeoutDefault: 45 s Seconds the browser waits for a page's initial load (page.goto()) to complete. Increase for very slow government sites.
JS Settle Timecrawler.js_settle_timeDefault: 3.0 s Extra seconds to wait after the page fires its load event, so JavaScript-rendered content (React, Angular) has time to populate the DOM before extraction runs. Increase to 5–6 s if JS-rendered pages return empty results.
Target Suffixescrawler.target_suffixesDefault: .gov.in  .nic.in Only follow links whose domain ends in one of these suffixes. Add .edu.in or .res.in to extend coverage to government-affiliated institutions.
Priority Keywordscrawler.priority_keywordsDefault: contact, officer, directory… URLs containing any of these words are crawled first. This ensures /contact, /staff, /directory pages are reached early rather than after thousands of generic navigation pages.
Skip Extensionscrawler.skip_extensionsDefault: .pdf, .jpg, .zip… URLs whose path ends in any listed extension are never queued. Prevents wasting worker slots on binary files and documents that contain no extractable email addresses.
Max Links per Depthcrawler.max_links_per_pageDefault: 100 / 50 / 40 / 15 Per-page link budget at each depth level (0 / 1 / 2 / default). Seed pages (depth 0) get 100 because home pages have many navigation links. Deeper levels get fewer since subpages are more focused. Tighter limits reduce runtime.
Enable Pagination-Aware Crawlingcrawler.pagination.enabledDefault: true Lets a genuine "Next page" link advance without spending Max Depth, so long paginated directories are crawled to completion. See Pagination-Aware Crawling above for how the safety detector works. Editable in Settings → Advanced.
Max Pages per Chaincrawler.pagination.max_pagination_pagesDefault: 50 Hard cap on how many "Next page" hops a single chain may follow, regardless of how many pages the site claims to have — the backstop if a chain is ever misjudged. Editable in Settings → Advanced.
Max Children per Chaincrawler.pagination.max_chain_childrenDefault: 100 Total ordinary (non-pagination) links the crawler will follow out of an ENTIRE chain, shared across all its pages combined — not per page. Prevents a 50-page chain from enqueuing 50× the usual per-page link budget. Editable in Settings → Advanced.
Anchor Text Signalscrawler.pagination.text_signalsDefault: next, », ›, more, last Link text treated as a pagination cue when the link carries no numbered page parameter to check instead. Shown read-only in Settings → Advanced; edit config.yaml directly to change this list.
Paging Query Paramscrawler.pagination.param_signalsDefault: page, pageno, start, offset, p Query parameter names checked for a page number (e.g. ?page=2). A recognized param with a non-numeric value (session tokens, junk) is treated as NOT pagination regardless of link text — this is the actual firewall against a disguised infinite-loop link. Shown read-only in Settings → Advanced; edit config.yaml directly to change this list.

Extraction Settings

Setting Description
Valid Email Suffixesextraction.email.valid_suffixesDefault: .gov.in .nic.in .res.in .ac.in Only emails whose domain ends in one of these suffixes are saved as leads. Emails from commercial providers found incidentally on government pages are discarded. Add .edu.in to include educational institutions.
Context Charactersextraction.email.context_charsDefault: 200 Characters captured on each side of a found email address. Stored as the "context snippet" in the lead record, and used to search for the person's name and title nearby.
Person Title Prefixesextraction.person.title_prefixesDefault: Shri, Dr, Mr, Smt, Ms, Prof… Words that signal the start of a person's name in the surrounding text. The crawler finds the nearest prefix followed by capitalized words and extracts that as the name.
Designation Keywordsextraction.person.designation_keywordsDefault: Secretary, Director, Commissioner… The first occurrence of any of these words near an email becomes the designation, followed by up to 60 more characters (e.g. "Joint Secretary, Ministry of Finance").
Proximity Charactersextraction.person.proximity_charsDefault: 300 Characters on each side of the email searched for name and designation matches. Increase to capture more context; decrease if unrelated text is being mistakenly extracted as names.

System Configs (Read-Only in UI)

These parameters appear in Settings → System Configs tab as read-only fields. To change them, edit portal/config.yaml directly and restart the server:

  • User Agent — The HTTP User-Agent header sent with every request.
  • Email Regex — The regular expression used to detect email addresses in page text.
  • JS Indicators — HTML strings that signal a page requires JavaScript rendering (e.g. <div id="__next").
  • Email Obfuscation Rules — Regex replacements for de-obfuscating disguised emails, e.g. [at] → @, (dot) → ..

Troubleshooting

  • No domains show in the table after import
    The import may still be running — check the progress bar in the sidebar. If it finished, the JSON file format may be invalid (must be an india.gov.in export). Try refreshing the page after the import completes.
  • Crawl job starts but finds zero leads
    Check three things: (1) The domain may not publicly list emails with .gov.in/.nic.in suffixes — check Valid Email Suffixes in Settings → Extraction Rules. (2) Max Depth may be too low — depth 0 only crawls the seed page; try 2 or 3. (3) The site may require JavaScript — enable Playwright Fallback in Settings.
  • JS-heavy site shows empty results / no content extracted
    Enable Playwright Fallback in Settings. Also try increasing JS Settle Time to 5–6 seconds to give React/Angular frameworks more time to finish rendering before extraction runs.
  • Crawl runs very fast but visits very few URLs
    Recrawl protection is likely skipping URLs already visited in a recent job. Go to Settings → System Configs and click Clear Cache to reset the visited-URL history, or reduce Recrawl Days to 0.
  • Dispatch stalls or a credential shows "Disabled"
    The SMTP server rejected the username or password (auth failure). Go to Settings → SMTP Credentials, edit the credential to fix the password, and re-enable it. For Gmail, ensure you are using an App Password — not your Google account password.
  • Campaign status stays PAUSED after clicking Dispatch
    No active credentials are available for sending. Check Settings → SMTP Credentials — all entries may be Disabled or Cooling. Wait for the cooldown to expire, fix a disabled credential, or add and test a new one.
  • Template renders with [MISSING: name] in the body
    The lead's person name is blank. Either (1) edit the name inline on the Leads page before creating the campaign, or (2) uncheck that specific draft in the campaign review so it is skipped during dispatch.
  • Sent count increases very slowly during dispatch
    This is expected. GovCrawler waits 30–90 seconds between sends (randomised jitter) to avoid SMTP rate limits. A campaign of 100 emails takes 50–90 minutes. Keep the app open and do not restart the server mid-dispatch.
  • A domain I want to crawl is not in the domain list
    The domain registry is sourced from india.gov.in. Run a Live API Refresh to fetch the latest additions. If the domain is not in the india.gov.in registry at all, it cannot be added through the standard import flow.
  • The bottom dock does not open after starting a crawl
    The dock is fixed to the bottom of the viewport — scroll down if it is not visible. If the job status immediately shows "failed", click on the job in the Recent Jobs sidebar to open the dock, then check the Server Logs tab for the error message.

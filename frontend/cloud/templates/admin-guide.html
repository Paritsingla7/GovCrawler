Admin Dashboard Guide

This is the cloud/VPS admin surface — it manages users, roles, and system oversight only. Crawling, leads, and campaigns are handled entirely on each operator's own machine via the agent launcher; they are not available here.

Users

Create a user with + New User, assign one of the three built-in roles (Admin, Operator, Viewer), and toggle their active status from the table. Permissions opens a per-user grant/deny grid for fine-tuning beyond the role default — leave a permission on "Inherited from role" unless you specifically need an exception for that one user.

"Admin" (the role) and "Super Admin" (the checkbox) are not the same thing. The Admin role is just a permission bundle — it happens to include every permission, but a per-user deny override can still take one away. Super Admin is a separate flag that bypasses every permission check unconditionally, with no override possible — reserve it for the handful of people who should never be locked out by a permissions mistake.

Roles

Roles are fixed (Admin/Operator/Viewer) and cannot be created or edited here — this tab is a read-only reference showing exactly what each role grants, so you know what you're assigning before you use the per-user permission overrides.

Audit Log

Every mutating admin action (user changes, permission overrides, password resets) is recorded here with the acting user, timestamp, and IP. Filter by an action prefix (e.g. user.) or a date range.

System

A quick health check: whether the database is reachable, which dispatch mode the SMTP sender is running in (embedded or external), and which agent machines have run a crawl job here recently.
